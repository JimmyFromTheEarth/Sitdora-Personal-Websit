• ECMAScript 和 JavasScript 的关系

    不同

    JavaScript 是由 NetScape 公司开发用于其浏览器产品的一个脚本语言。 并且由 JavaScript 这一商标保护。
    ECMAScript 是指由 ECMA 标准化组织发行的一个国际通用的脚本语言标准， 一句此标准实现的脚本语言都可称为 ECMAScript 。

    关系

    ECMAScript 这个标准最初是由 NetScape 公司将其脚本语言 JavaScript 提供给国际标准化组织作为标准制定的参考，由 ECMA 在此基础上开发出一个国际通用的标准。
    因此， JavaScript 是　ECMAScript 标准的一种实现。
    在一般情况下， JavaScript 和 ECMAScript 是可互换的。

    其他信息

    按照 ECMAScript 这一标准实现的脚本语言还有：Jscript，ActionScript ...

  • ES6 与 ECMAScript 2015

    ES6 表示一个版本：ECMAScript 6
    而ECMAScript 2015 指代版本发行的年份： ECMAScript 6.0 - 2015/6

    版本发布历史

    • 06/1997 - ECMAScript 1.0 (Release)
    • 06/1998 - ECMAScript 2.0 (Release)
    • 12/1999 - ECMAScript 3.0 (Release)
    • 00/2000 - ECMAScript 4.0 (Starts to Proposal)
    • 00/2007 - ECMAScript 4.0 (Draft)
    • 00/2008 - ECMAScript 4.0 (Abandoned, splits to 3.1 and .next)
    • 12/2009 - ECMAScript 5.0 (Release, 原 ECMAScript 3.1)
    • 06/2011 - ECMAScript 5.1 (Release)
    • 00/2013 - ECMAScript 6.0 (Draft)
    • 00/2015 - ECMAScript 6.0 (Release)
    • 06/2016 - ECMAScript 6.1 (Release)
    • 06/2017 - ECMAScript 6.2 (Release)
    • ...
    **00/2009 中的00表示月份未知

  • 转换 ES6 风格的代码 到 ES5

    Babel

    • 配置 .babelrc 文件， 放于项目根目录下。
                                  {
                                "presets":[],
                                "plugins":[]
                            }
      presets 可取的值来自于安装的 Babel 转码规则, 一下列举规则的安装命令， 取所需。 
                                  $ npm install --save-dev babel-preset-latest
                            $ npm install --save-dev babel-preset-react
                            $ npm install --save-dev babel-preset-stage-0
                            $ npm install --save-dev babel-preset-stage-1
                            $ npm install --save-dev babel-preset-stage-2
                            $ npm install --save-dev babel-preset-stage-3
      Babel 转码工具 babel-cli, 可安装在全局或者项目下， 一般选择后者。 方便每个项目使用不同的 .babelrc 文件。
                                  $ npm install --global babel-cli ## for global
                            $ npm install --save-dev babel-cli ## for project
      执行 babel-cli 
                                  ## global model
                            $ babel example.js ## the complie result go to standard output stream
                            $ babel example.js --out-file/-o complied.js ## use --out-fiel / -o offer a file for output 
                            $ babel src-lib --out-dir/-d lib ## use --out-dir/-d offer a directory for output
                            $ babel src -d lib -s ## complied entier libarray with map file
                            ## project model
                            ## first to update the package.json file
                            ## to 
                            {
                                //..
                                "devDependencies":{
                                    "babel-cli":"^6.0.0"
                                },
                                "scripts":{
                                    "build":"babel src -d lib"
                                },
                            }
                            ## the use the command below in the project directory
                            $ npm run build

    • babel-node

      babel-node 是 babel-cli 自带的命令， 可用于替代 node 命令， 先转码到 ES5 再进行 node 运行。

    • babel-register

      这一命令用于改写 require(), 在加载.js/.jsx/.es和.es6的文件时首先转码再进行 requier。

    • babel-core

      用于在代码中进行转码。 
                                      ## the code can call APIs to complie
                                $ npm install babel-core --save

                                ## example to use
                                var babel = require("babel-core");
                                babel.transform('code here', options);
                                babel.transformFile('file.js', options, function(err, result){});
                                babel.transformFileSync('file.js', options);
                                babel.transformFromAst(ast, code, options);
      options 的配置参考官方文档

    • babel-polyfill

      用于转换新的 API, Babel 默认只会转换新语法。 具体请参考官方文档
